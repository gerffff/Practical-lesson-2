# Призначення бібліотеки
Ця бібліотека є навчальним прикладом поступової еволюції коду від простих функцій до повноцінного пакета з налаштованими інструментами перевірки якості. У ній реалізовано базові утилітарні функції (add, capitalize, formatNumber, groupBy) та клас Logger для роботи з повідомленнями різних рівнів.

Мета бібліотеки — продемонструвати використання TypeScript-типізації, перевірок ESLint/Prettier, Husky-хукiв і семантичного версіонування. Вона показує, як будувати та розвивати код так, щоб він залишався чистим, зрозумілим і стабільним, а також як правильно оформлювати зміни у репозиторії.

### Інструкції з запуску npm i, npm run demo, npm run build: 
### npm i
Команду вводимо у терміналі, у кореневій папці проєкту, де лежить package.json. Команда встановлює всі залежності, перелічені у файлі package.json. <br>
Вводимо вісля клонування репозиторію з GitHub або отримання проєкту від іншого розробника
### npm run demo
Вводимо у терміналі після встановлення залежностей після того, як проєкт уже налаштований і всі залежності встановлені (npm i виконано). <br>
Команда потрібна для щоб перевірки того, як працюють написані функції або класи.

### npm run build
Вводимо у термінал після завершення розробки або перед публікацією/відправленням коду в репозиторій.
Генерує з TypeScript готовий JavaScript-код

## Огляд кроків еволюції

0.1.0 — створено базові функції з типом any, перевірено роботу лінтера.
Це перша мінімальна робоча версія — створено базовий код без стабільного API. Використано any, тому версія починається з 0.1.0 (етап активної розробки, а не 1.0.0).

0.2.0 — додано базові типи number і string, перевірено типізацію.
Зміни суттєво вплинули на структуру типів функцій, але backward-compatibility (сумісність) не порушено.
Тому збільшено другу цифру (minor) — з 0.1.0 → 0.2.0.
Якби були лише дрібні виправлення без нової логіки, версію підняли б до 0.1.1, але тут з’явилась нова функціональність (типізація).

0.3.0 — додано функцію formatNumber із type alias та опціональними параметрами.
Було додано нову функцію, що розширює API бібліотеки, тому логічно збільшено minor-версію (0.2.0 → 0.3.0).
Немає змін, що ламають попередній код, тому major не підвищувався.

0.4.0 — реалізовано інтерфейс User та generic-функцію groupBy.
З’явилася ще одна нова суттєва функціональність, але API сумісне.
Тому версію піднято знову на minor, а не patch — адже додано нові можливості, а не просто виправлено помилки.

0.5.0 — створено клас Logger, підключено .env і валідацію через zod.
Додано новий великий компонент (Logger) і роботу з конфігурацією середовища — це значне розширення бібліотеки.
Але без порушення сумісності, тому лише minor зростання.

1.0.0 — стабілізовано публічний API, заборонено any, додано exports у package.json.
Це перший стабільний реліз. Після серії нульових версій (0.x.x) бібліотека вважається готовою для використання,
тому версія піднята до 1.0.0 (перший major) — початок стабільного циклу.

2.0.0 — змінено сигнатуру add (breaking change).
Функція add тепер приймає масив (number[]) замість кількох аргументів — це несумісна зміна з попередньою версією.
Тому підвищено major-номер (1 → 2) за правилами SemVer, адже старий код перестане працювати без змін.
